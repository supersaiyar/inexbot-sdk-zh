# 程序操作

### 程序打开

- 说明
- 正式执行程序之前需要打开程序
- 上位机下发，打开程序
**命令字：** `0x5100`
- "robot":1
- 表示要操作的机器人编号；int类型，取值范围[1,4]
- "jobName":"xxxx"
- 新建作业文件名；string类型
- "suffixname":".JBR"

```json
{
  "robot":1,
  "jobName":"xxxx",
  "suffixname":".JBR"
}
```
### 程序复制

- .JBR前台程序  .JBP后台局部程序  .JBPG后台全局程序
- 说明
- 复制后程序的后缀是一样的
- 上位机下发复制程序
- 命令字：5101
- "robot":1
- 表示要操作的机器人编号；int类型，取值范围[1,4]
- "oldjobname":"QQQQ"
- 旧文件名；string类型
- "newjobname":"WWWW"
- 复制后文件名；string类型
- "suffixname":".JBR"
- 文件后缀名；string类型

```json
{
  "robot":1,
  "oldjobName":"QQQQ",
  "newjobName":"WWWW",
  "suffixname":".JBR"
}
```
### 程序清除

- .JBR前台程序  .JBP后台局部程序  .JBPG后台全局程序
- 说明：
- 会清除当前机器人的所有程序，但不会清除复位程序
- 上位机下发，清空当前机器人作业文件
**命令字：** `0x5102`
- "robot":1
- 表示要查询的机器人编号；int类型，取值范围[1,4]
- "lock_enable":true

```json
{
  "robot":1,
  "lock_enable":true
}
```
### 获取当前打开程序

- 是否清空作业文件使能；bool类型
- true：清空使能；false：清空不使能
- 上位机下发，获取当前打开程序
**命令字：** `0x5103`
- "robot": 1

```json
{
  "robot":1
}
```
- 表示要操作的机器人编号；int类型，取值范围[1,4]
- 控制器收到0x5103命令后，发送以下命令
**命令字：** `0x5104`
- "openedJobName":"abs"

```json
{
  "openedJobName":"asd"
}
```
### 主程序

- 当前打开的作业文件；string类型
- 说明：
- 主程序可以认为是特殊的前台程序，只有前台程序可以设置成主程序
- 上位机下发，设置作业文件为主程序
**命令字：** `0x5110`
- "robot":1
- 表示要查询的机器人编号；int类型，取值范围[1,4]
- "jobName":"xxx"

```json
{
"robot":1,
"jobName":"xxx"
}
```
- 空表示没有；string类型
- 当取消主程序时，该字符串为空
- 上位机下发，查询主程序
**命令字：** `0x5111`
- "robot":1

```json
{
  "robot":1
}
```
- 表示要查询的机器人编号；int类型，取值范围[1,4]
- 控制器回复主程序查询结果
**命令字：** `0x5112`
- "robot":1
- 表示要查询的机器人编号；int类型，取值范围[1,4]
- "jobName":"xxx"

```json
{
  "robot":1,
  "jobName":"xxx"
}
```
### 程序保存

- 空表示没有；string类型
- 说明：
- 程序进行修改、插入、删除后需要使用该协议进行保存
- 上位机下发，保存程序
**命令字：** `0x5120`
- "robot":1
- 表示要查询的机器人编号；int类型，取值范围[1,4]
- "jobname":"xxxx"
- 作业文件名；string类型
- "suffix":".JBR"
- .JBR前台程序  .JBP后台局部程序  .JBPG后台全局程序
- "time":"2024.08.08 14:16:39"

```json
{
  "robot":1,
  "jobname":"xxxx",
  "suffix":".JBR",
  "time":"2024.08.08 14:16:39"
}
```
### 同步程序打开（功能可能有问题，只有在6001打开时同步到6000的操作）

- 时间戳
- 说明：
- 控制器打开程序后通知上位机，告知上位机该程序已在控制器打开
- 控制器发送给上位机
**命令字：** `0x3130`
- "robot":1
- 表示当前反馈的机器人编号；int类型，取值范围[1,4]
- "jobName":""

```json
{
  "robot":1-4,
  "jobName":""
}
```
### 发送程序指令

- 作业文件名；string类型
- 说明：
- 为无作业文件的指令插入
- 运行方式与作业运行方式一样，但是需要将"noFileRun"节点的值写为true
- 上位机下发插入指令
**命令字：** `0x5122`
- "robot":1
- 表示要操作的机器人编号；int类型，取值范围[1,4]
- "cmd":cmd->toJson()

```json
{
  "robot":1,
  "cmd":cmd->toJson()
}
```
### 程序同步

- 表示插入的指令详情，见指令页面；object类型
- 上位机发送给控制器,通知示教器同步作业文件
**命令字：** `0x1222`
- data:无
- 控制器响应给上位机示教器的作业文件同步结果
**命令字：** `0x1223`
- "finished":true

```json
{
  "finished":true
}
```
- true:成功；false:失败
- 控制器通知示教器同步作业文件
**命令字：** `0x1224`
- data:无
- 示教器响应给控制器作业文件同步结果
**命令字：** `0x1225`
- "finished":true

```json
{
  "finished":true
}
```
- true:成功；false:失败
```
