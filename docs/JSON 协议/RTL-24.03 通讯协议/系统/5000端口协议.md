# 5000端口协议

- 说明：
- 5000端口协议主要作用于上传、下载、备份文件。
- 若要向控制器上传文件，则上位机通过0x1D00请求上传文件，控制器通过0x1D01回复上传请求，当控制器同意上传时，上位机通过0x1D04上传文件，上传完毕后，上位机通过0x1D05告诉控制器上传结束，控制器通过0x1D06回复确认接收成功。
- 若要下载控制器中的文件，则上位机通过0x1D01请求下载文件，控制器通过0x1D03回复下载请求，当控制器同意下载时，控制器通过0x1D07向上位机传输其申请下载的文件，并且上位机通过0x1D08回复控制器接收结果，下载完毕后，控制器通过0x1D09告诉上位机下载结束，上位机通过0x1D0A回复确认接收成功。
- 上位机请求上传文件
**命令字：** `0x1D00`
- "name": "vxWorks"
- 上位机发送文件的文件名；string类型
- "size": 4096

```json
{
"name":"vxWorks",
"size":4096
}
```
- 上位机发送文件的文件大小；int类型
- 控制器回复上传请求
**命令字：** `0x1D01`
- "answer":"yes"

```json
{
"answer":"yes"
}
```
- 是否同意上传文件请求；string类型
- "yes": 同意上传文件请求，"busy": 控制器忙碌
- 上位机请求下载文件
**命令字：** `0x1D02`
- "name": "job/R1/WWWWW.JBR"

```json
{
"name":"job/R1/WWWWW.JBR"
}
```
- 需要下载的作业文件名称；string类型
- 参数name是作业文件在控制器中的作业文件路径加上名字。在0x5533和0x5534中系统有向上位机发送路径和名字。示教器提示开始同步作业文件。
- 控制器回复下载请求
**命令字：** `0x1D03`
- "answer": "yes"
- 系统是否同意下载文件请求；string类型
- "yes":  同意下载文件请求，"busy":  控制器忙碌
- "name": "job/R1/WWWWW.JBR"
- 需要下载的作业文件名称；string类型
- "size": 217

```json
{
"answer":"yes"
"name":"job/R1/WWWWW.JBR"
"size":217
}
```
- 此作业文件的文件大小；int类型
- 上位机向控制器上传文件
**命令字：** `0x1D04`
- 上位机告诉控制器上传结束
**命令字：** `0x1D05`
- 控制器回复确认接收成功
**命令字：** `0x1D06`

```json
{
"finish": true
}
```
- "finish"：true
- 判断文件是否上传成功；bool类型
- true: 上传成功，false: 上传失败
- 控制器向上位机发送申请下载的文件
**命令字：** `0x1D07`
- 上位机回复接收结果
**命令字：** `0x1D08`
- 控制器告诉上位机下载完成
**命令字：** `0x1D09`
- 上位机回复确认接收成功
**命令字：** `0x1D0a`

```json
{
"finish": true
}
```
- "finish"：true
- 判断文件是否下载成功；bool类型
- true: 下载成功，false: 下载失败
- 下载连接断开
**命令字：** `0x1D0b`
- 上位机请求备份文件
**命令字：** `0x1D10`
- 控制器回复备份请求
**命令字：** `0x1D11`
- "answer":"controller.tar.gz"

```json
{
"answer":"controller.tar.gz"
}
```
- 回复备份文件名；string类型
- 上位机发送下载log文件完成
**命令字：** `0x1D12`
- 上位机发送下载eni文件完成
**命令字：** `0x1D13`
