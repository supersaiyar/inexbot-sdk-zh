# 7000端口协议

### 7000端口设置

| 项目 | 长度(Byte) | 说明 |
|---|---|
| SyncByte | 2 | 0x4E66固定包头 |
| Length | 2 | 16进制长度0-0xffff;不包含数据头和命令字，不包含CRC段 |
| Command | 2 | 命令字 |
| data | Length | 数据段（主要为JSON数据格式） |
| CRC | 4 | 对不含包头数据做CRC校验(CRC32) |

- 说明：
- 7000端口可以选择”TCP”和”UDP”两种通讯协议。可以通过进入控制器后台修改控制器配置选择那种通讯协议。在控制器配置文件config/controller.json中，”hostComputerService”节点下：”socketType”节点表示socket类型，可选择”TCP”和”UDP”，默认为”TCP”；”checkCRC”节点表示是否进行CRC检测，选择false时，不检测CRC，但注意CRC所占用的4个字节的数据仍要发送；默认为true，检测CRC。查询状态可以设置为“UDP”通讯进行数据获取，最小周期可以达到1ms。UDP demo下载。
- 查询状态
**命令字：** `0x1E00`
- "channel": 1
- 通道； int类型
- 最多9个通道，默认为1
- "stop":1
- 停止发送；int类型
- 为1时,停止之前的持续发送,下面的数据可不发
- "robot": 1
- 表示机器人编号；int类型
- 取值范围[1,4]
- "mode": 0
- 请求回复模式；int类型
- 0：应答(请求一次回复一次)，1：持续(请求一次持续回复)
- "interval":10,
- 持续回复间隔时间；int类型
- 单位为ms，仅mode为持续时有效，范围[1, 60000]
- "queryType":  [ "realPosACS","realPosMCS","realPosPCS","realPosUCS", "axisVel", "axisAcc", "torque", "electric", "numGVar", "IO","serialNum""runControlPosRunNum","detailedMotionPos" ],
- 查询运行参数，string类型
- 无需查询的不写即可
- "realPosACS"：实时坐标位置（关节坐标系）
- "realPosMCS"：实时坐标位置（直角坐标系）
- "realPosPCS"：实时坐标位置（工具坐标系）
- "realPosUCS"：实时坐标位置（用户坐标系）
- "axisVel"：轴速度
- "axisAcc"：轴加速度
- "torque"：力矩
- "electric"：电流
- "numGVar"：数值变量
- "IO"：IO端口
- "serialNum"：序列号
- "runControlPosRunNum"：运动控制当前轨迹已运行了几个点位
- "detailedMotionPos"：详细运动点位，该项每个机器人同一时刻仅支持一个通道查询
- "typeCfg":
- 查询内容配置，仅需要查询相关类型数据时有效
- "numGVar": ["GD001", "GA001", "GI001", "GI333" ]
- 查询全局变量；int类型。
- "IO":[ “DI1”, “DI16”, “DO1”, “DO3”, “DO17”]
- 查询IO；string类型
- 可查询的最大数量不应大于IO个数
- 若查询的IO不存在，返回-1
- "detailedMotionPos"
- 查询更详细的运行参数
- "num":10
- 每帧数据回复的点位数目；int类型
- "optional": ["ACS", "MCS", "time", "reset"]

- 控制器返回查询结果
**命令字：** `0x1E01`
- "channel":1

- "electric"：[ 1.1, 1.2, 1.3, 1.4, 1.5, 1.6, 1.7, 2.1, 2.2, 2.3, 2.4, 2.5 ]

- "detailedMotionPos"
- 回复更详细的运行参数

- 每帧数据回复的点位数目；int类型

- 回复选择参数，string类型

- "ACS": [ 1.1, 1.2, 1.3, 1.4, 1.5, 1.6, 1.7, 2.1, 2.2, 2.3, 2.4, 2.5 ]

- 每一个点位的时间戳，第一个数为秒，第二个数为纳秒
- 运动控制
- 命令字：1E02
- "robot": 1
- 表示机器人编号，int类型
- 取值范围[1,4]
- "clearBuffer": 1
- 清空运动轨迹缓存队列，int类型
- 为1时，清空运动轨迹缓存队列,下面的数据可不发，若发送下面的数据，清空运动轨迹缓存队列时，不影响当前正在传输的数据若为0，可不发“clearBuffer”
- "targetMode": 0,
- 运动模式；int类型
- 0：独立点位，1：连续轨迹
- "extMove": 0,
- 是否有外部轴运动；int类型
- 0或不写：无外部轴运动，1：有外部轴运动
- "sync": 0
- 外部轴是否同步；int类型
- 0或不写：外部轴不同步，1：外部轴同步
- "speed": 100,
- 速度设定；int类型
- 若不写，则使用默认值
- "acc": 100
- 加速度设定；int类型
- 若不写，则使用默认值
- "pl": 5
- 若不写，则使用默认值5；仅"独立点位"、使用"targetVec"节点时有效
- "moveMode":"TeachSet"
- 运动模式；string类型
- 可选"TeachSet","MOVJ","MOVL","MOVS"，默认为"TeachSet"
- 仅“独立点位”、使用“targetVec”节点时有效
- "targetPos":[ 1.1, 1.2, 1.3, 1.4, 1.5, 1.6, 1.7, 2.1, 2.2, 2.3, 2.4, 2.5 ],
- 目标点，若使用该节点则只会走一个目标点
- "targetVec":
- 多目标点，若使用该节点则会按顺序走完所有点位
- "pos":[ 1.1, 1.2, 1.3, 1.4, 1.5, 1.6, 1.7, 2.1, 2.2, 2.3, 2.4, 2.5 ]},

- 运动控制的回复
**命令字：** `0x1E03`
- "robot": 1
- 表示机器人编号，int类型
- 取值范围[1,4]
- "success": false,
- 判断是否成功， bool类型
- true：成功；false：失败
- "size": 1
- 运动轨迹缓存队列的当前轨迹数目，int类型
- 仅当"success"为true时有该项
- "cause": "timeout"

```json
{
"robot":1
"success":false,
"size":1,
"cause":"timeout"
}
```
- 接收失败的原因，string类型
- 接收成功时为空，”busy”: 当前有未传输完成的数据，”timeout”: 接收超时，”dataErr”：数据错误，”termination”: 发送端终止了正在传输的数据
- 变量、IO控制
**命令字：** `0x1E04`
- "numGVar": {"GA001": true, "GI033": 6, "GD666": -2.33}
- 查询变量， string类型
- "GA001","GI033","GD666"皆为变量名，其中GA为bool型变量，GI为int型变量，GD为Double型变量
- "IO": {"DO1":1, "DO13":0}
