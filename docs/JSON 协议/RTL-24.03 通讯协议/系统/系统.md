# 系统

### 状态

- 说明：
- 系统开机需要进行初始化，否则系统将无法使用。
- 上位机查询是否控制器初始化是否完成
**命令字：** `0x1000`
- date：无；
- 控制器回复查询结果
**命令字：** `0x1001`
- "finishinit": true

```json
{
  "finishinit": true
}
```
- 初始化是否完成；bool 类型
- true：完成，false：未完成
- 控制器是否为T5系统查询
**命令字：** `0x1002`
- date：无
- 控制器回复查询结果
**命令字：** `0x1003`
- "T5_system":

```json
{
  "T5_system":false
}
```
- 是否为T5系统 ；bool 类型
- true：是，false：不是
- 控制器硬件系统发布版本查询
**命令字：** `0x1004`
- date：无
- 控制器回复硬件系统发布版本查询结果
**命令字：** `0x1005`
- "result": true
- 查询是否成功；bool 类型
- true：是，false：否
- "version": xx

```json
{
 "result":true,
 "version":xx
}
```
### 版本升级

- 版本号内容；string 类型
- 若查询成功，则返回版本号
- 说明：
- 系统开机时需要查询当前版本号，若出现不匹配，则需要进行版本升级
- 上位机查询当前系统版本号
**命令字：** `0x1010`
- "version": "v1.0-rc1-67-gf34dae7"

```json
{
  "version": "v1.0-rc1-67-gf34dae7"
}
```
- 版本号内容；string 类型
- 控制器回复查询结果
**命令字：** `0x1011`
- "configFileVersionMismatch": false
- 配置文件是否匹配；bool 类型
- false：不匹配，true：匹配
- "jobFileVersion": "5.2.0"
- 作业文件版本；string 类型
- "rtlVersion": "22.08.0"
- 控制器版本；string 类型
- "sysClock": "2023.11.10 13:29:25"
- 系统时钟，string 类型
- "version": "v1.0-rc1-67-gf34dae7"
- 版本号内容；string 类型
- 上面3402发送的字段如果是个空字符串的话返回的数据也还是同样的内容

```json
{
  "configFileVersionMismatch": false,
  "jobFileVersion": "5.2.0",
  "rtlVersion": "22.08.0",
  "sysClock": "2023.11.10 13:29:25",
  "version": "v1.0-rc1-67-gf34dae7"
}
```
### 日志

- 控制器节点类型示例：https://ones.inexbot.com/wiki/#/team/RnqpQ1Yp/page/B64xwupH?blockId=nPUaBaWTx
- 说明：
- 获取日志文件和系统崩溃日志文件
- 上位机查询崩溃日志文件
**命令字：** `0x1020`
- date：无
- 控制器回复崩溃日志文件
**命令字：** `0x1021`
- "absolutepath":
- 路径；string 类型
- "dumpLogfilenum":
- 文件数量；int类型
- "dumpLogfilelist": ["","","",""]

```json
{
"absolutepath":"",
"dumpLogfilenum":3,
"dumpLogfilelist":["","","",""]
}
```
- 日志文件列表；string类型
- 上位机请求获取日志文件列表
**命令字：** `0x1022`
- "num": 5

```json
{
"num":5
}
```
- 表示获取最近多少个文件；int 类型
- 可取5、30、100
- 控制器回复日志文件列表
**命令字：** `0x1023`
- "absolutepath": "log/"
- 日志所在目录；string 类型
- 可取5、30、100
- "logfilenum": 3
- 日志数目；int 类型
- 这个数和上面的num不一定相等
- "logfilelist": ["logInfo.0","logInfo.1","logInfo.2"]

```json
{
"absolutepath":"log/",
"logfilenum":3,
"logfilelist":["logInfo.0","logInfo.1","logInfo.2"]
}
```

### 时间设置

- 控制器dns；string类型
- 上位机发送时间查询
**命令字：** `0x1050`
- "type": 0
- 1: 获取系统开始时间至现在秒数，0: 现实时间，double 类型
- "format": 0

```json
{
  "type": 0,
  "format": 0
}
```
- 1: 获取北京时间，0: 获取时间戳，double 类型
- 控制器回复查询结果
**命令字：** `0x1051`
- "tType":0
- 1: 获取系统开始时间至现在秒数，0: 现实时间；int 类型
- "format":0
- 1: 获取北京时间，0: 获取时间戳；int 类型
- "date":
- 包含日期和时间信息，分别记录年、月、日、时、分、秒、毫秒等值，通常用于表示某一时刻的完整时间信息。
- "year":2024
- 日期中的年份；int 类型
- mon":1
- 日期中的月份；int 类型
- "day":1
- 日期中的天；int 类型
- "hour":1
- 时间中的小时；int 类型
- "min":1
- 时间中的分钟；int 类型
- "sec":
- 时间中的秒数；int 类型
- "msec":1
- 时间中的毫秒数；int 类型
- "timestamp":
- 以秒和纳秒的形式记录时间戳，用于高精度的时间计算，通常与日期结合使用。
- format为0时显示。
- "s":1
- 时间戳中的秒数；int 类型
- "ns":1

```json
{
  "tType": 0,
  "format": 0,
  "date": {
    "year": 2023,
    "mon": 1,
    "day": 1,
    "hour": 1,
    "min": 1,
    "sec": 1,
    "msec": 1
  },
  "timestamp": {
    "s": 1,
    "ns": 1
  }
}
```
- 时间戳中的纳秒数；int 类型
- 说明：
- 查询系统支持的语言，并显示当前使用的语言。
- 上位机查询语言支持
**命令字：** `0x1052`
- data：无
- 控制器回复当前语言支持
**命令字：** `0x1053`
- "language" : ["Chinese","Korean","English"……]
- 控制器能够支持的所有语言，string 类型
- 目前能够支持中文，韩文，英文。
- "language_used" : "Chinese"

```json
{
"language" : ["Chinese","Korean","English"]
"language_used" : "Chinese"
}
```
- 控制器当前使用的语言；string 类型
- 查询上位机当前语言
**命令字：** `0x1054`
- "teachbox_language" : "Chinese"

```json
{
"teachbox_language" : "Chinese"
}
```
