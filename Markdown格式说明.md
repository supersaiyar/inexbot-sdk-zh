# 1. 文档概述

本文档旨在演示 **Markdown** 的核心语法特性，同时展示如何通过**元数据**优化文档结构，使其更易于被 AI 模型检索和理解。

> **提示**：在构建 AI 知识库时，清晰的层级结构和准确的元数据标签能显著提高检索命中率。

## 1.1 为什么需要元数据？

元数据（Metadata）是“关于数据的数据”。对于 AI 而言，它提供了上下文信息：
*   **分类**：快速定位文档所属领域。
*   **时效性**：通过 `date` 字段判断信息是否过时。
*   **关联**：通过 `tags` 建立文档间的语义连接。

# 2. 文本格式化与引用

Markdown 支持丰富的文本样式，用于强调重点内容。

## 2.1 基础样式

我们可以使用 **粗体** 来强调重点，使用 *斜体* 来表示术语，或者使用 `行内代码` 来展示命令。

如果需要大段引用官方文档或他人观点，可以使用引用块：

> “任何足够先进的科技都与魔法无异。”
>
> —— 亚瑟·克拉克 (Arthur C. Clarke)
>
> 引用块可以包含多段文字，并且支持嵌套。
> > 这是嵌套的引用内容，通常用于回复中的回复。

## 2.2 列表展示

### 无序列表
无序列表常用于列举没有特定顺序的项目：
*   🍎 苹果
*   🍌 香蕉
*   🍇 葡萄
    *   巨峰葡萄
    *   夏黑葡萄
*   🍊 橙子

### 有序列表
有序列表适用于步骤说明或排名：
1.  **第一步**：打开终端。
2.  **第二步**：输入命令 `npm install`。
3.  **第三步**：运行项目 `npm start`。
    1.  检查端口占用。
    2.  确认服务启动成功。

# 4. 数据表格

表格是展示结构化数据的最佳方式。以下是一个包含对齐方式的示例表格：

| 功能特性 | 支持状态 | 优先级 | 备注 |
| :--- | :---: | :--- | :--- |
| 多级标题 | ✅ 已支持 | 高 | 必须包含 H1-H3 |
| 表格渲染 | ✅ 已支持 | 中 | 支持对齐控制 |
| 数学公式 | ⚠️ 部分支持 | 低 | 需开启插件 |
| 流程图 | ❌ 未支持 | - | 暂不纳入范围 |

*   `:---` 表示左对齐
*   `:---:` 表示居中对齐
*   `---:` 表示右对齐

# 5. 代码块示例

除了行内代码，我们还可以展示多行代码块，并指定语言以获得语法高亮：

```python
def calculate_ai_score(metadata):
    """
    根据元数据计算文档的 AI 检索得分
    """
    score = 0
    if 'tags' in metadata:
        score += len(metadata['tags']) * 10
    if metadata.get('version') == '1.0.0':
        score += 5
    return score

# 调用函数
doc_meta = {"tags": ["ai", "md"], "version": "1.0.0"}
print(calculate_ai_score(doc_meta))
```
